Análise Comparativa: Healthmap x Tabia (CareOS)

Cruzamento: mapeamento CBV x Healthmap (19 endpoints) + OpenAPI CareOS — 13 abr 2026

1. Resumo Executivo ▼
19Endpoints Healthmap
8Equivalente direto
8Equivalente parcial

A Tabia cobre a maioria das necessidades, porém com abordagem arquitetural diferente — especialmente nos domínios de Plano de Ação (Pathway/ICP/Task/Event) e Questionários (Survey/SurveyData). A migração exigira camada de adaptação e mapeamento de campos significativo.

2. Mudancas Estruturais Importantes ▼

Mecanismo

API Key fornecida pela Tabia, enviada em cada request como header. Não há fluxo OAuth2.

Header esperado (todas as chamadas)

Diferença: Elimina completamente o fluxo de obtencao/renovacao de token. A API Key é estática por ambiente — não expira como o Bearer Token.
Migrar

Remover toda logica de client_credentials. Substituir Authorization: Bearer + KeyDS por header api-key.
